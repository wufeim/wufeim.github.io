Abstract

Writing code is fun but writing bug is not. The causes of some bugs are not clear, especially when the error messages are misleading.

1 Plotting with OpenCV

Error messages in OpenCV can be confusing and misleading sometimes. When I was debugging my PyTorch dataloader, I tried to plot certain keypoints in the image. 

img = sample['img'][0].permute(1, 2, 0).cpu().numpy()
img = cv2.circle(img, kp, 1, (0, 255, 0), -1)
I got this error message: 
cv2.error: OpenCV(4.5.2) :-1: error: (-5:Bad argument) in function 'circle'
> Overload resolution failed:
>  - Layout of the output array img is incompatible with cv::Mat (step[ndims-1] != elemsize or step[1] != elemsize*nchannels)
>  - Expected Ptr for argument 'img'
The reason is that OpenCV plotting requires the image array to be contiguous in memory, and the fix is simple: 
img = np.ascontiguousarray(img, dtype=np.uint8)
